# Claimrail

Claimrail is a private eligibility rail for **private lending and borrowing on Solana**.

Borrowers submit an encrypted dossier. Lenders define policy. Arcium computes a minimal eligibility outcome. Verifiers reveal only policy-approved claims.

Live app: [https://claimrail.vercel.app](https://claimrail.vercel.app)

The current submission target is **Arcium RTG** only, specifically:

- RTG: [Private Lending & Borrowing](https://rtg.arcium.com/rtg/dev-lending-borrowing)
- core thesis: lending should not expose borrower dossiers, collateral posture, or underwriting inputs in public state

## Product framing

Claimrail is not presented as a generic KYC form.

The product wedge is:

- applicant / borrower submits a private dossier
- issuer / lender defines underwriting and compliance policy
- Arcium computes a minimal eligibility outcome privately
- verifier sees only policy-approved claims when selective reveal is required

The intended outcome is:

```text
encrypted borrower inputs + lender policy
→ Arcium confidential computation
→ minimal lending eligibility outcome
→ scoped reveal only if needed
```

## RTG-aligned use case

For the Arcium RTG `Private Lending & Borrowing` track, Claimrail should be understood as:

- private borrower onboarding for lending venues
- encrypted policy evaluation for borrower eligibility
- selective reveal for compliance, servicing, or transfer-agent style review
- reusable attestations instead of broad dossier exposure

Example outcome surface:

- eligible / not eligible
- risk tier
- borrowing cap
- manual review required

## Build / test / deploy checklist

```bash
# install app deps
cd app && npm install

# install/update Arcium tooling
arcup self update
arcup update

# build confidential instructions + program
arcium build

# local tests
arcium test

# devnet deploy
arcium deploy \
  --cluster-offset 456 \
  --recovery-set-size 4 \
  --keypair-path ~/.config/solana/claimrail-arcium.json \
  --rpc-url https://your-devnet-rpc
```

After deployment, initialize computation definitions:

```bash
node --loader ts-node/esm scripts/init-computation-defs.ts
```

Then run remote tests:

```bash
arcium test --cluster devnet
```
